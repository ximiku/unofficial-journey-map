# 设计说明

## 概念

玩家受雇于一个暧昧的整理机构，表面任务是绘制清楚的取经路线图。系统真正追问的是：清楚的地图如何生产省略，旁注如何抵抗省略，空白何时不是缺失而是一种伦理。

## 主循环

1. 从首页进入地图。
2. 在连续取经路上滑动远望后续地点，但只能进入当前或已完成节点。
3. 每站按“抵达、线索、批注、审稿 / 互动、总结”推进。
4. 选择批注放置区，或处理审稿 / 互动模块，数值随之改变。
5. 完成本站总结，自动进入下一站。
6. 结束旅程，依据数值、访问节点和批注类型解析结局。

## 系统

- `annotationEngine` 负责批注放置和数值变化。
- `reviewEngine` 负责审稿事件触发和审稿选择。
- `endingResolver` 负责多结局解析。
- `persistence` 负责版本化 localStorage 存档。
- `completedNodeIds` 记录已经走完流程的节点，控制线性解锁。

## 美术方向

视觉以旧纸、墨色、朱砂、蓝黑批改和灰烬色为主。界面避免远程素材和大型插画，但允许更明显的游戏反馈：卷轴边缘、朱印状态、线绳、纸片、地图裂痕、小妖档案员提示和余声支线。文本阅读优先，但不把界面做成纯资料后台。

## UX

开始旅程后的首屏由三部分组成：连续滑动地图、当前任务摘要、新任档案员交接单。地图下方没有 1-26 跳转索引；滑条只移动视野，不改变当前节点。节点分类不再用上方框式筛选，而是直接体现在地图点颜色、形状、纹理和尺寸中，并通过小字图例说明。下方工作台是地点流程卡，主流程只显示综合故事和少量选择；完整资料留在档案馆。所有主要控件是按钮或表单控件，可键盘访问；数值标签使用 `title` 和说明文字辅助理解。
