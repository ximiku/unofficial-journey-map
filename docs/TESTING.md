# 测试策略

## 命令

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

## 覆盖内容

- 结局解析器。
- 审稿触发与选择。
- 批注放置效果。
- localStorage 存档序列化与恢复。
- 数据引用完整性。
- 首页、连续地图滑条、地图图例、地点流程、引导、档案馆组件基础渲染。

## 手动 QA

1. 开始新游戏。
2. 确认“新任档案员交接单”说明按地点流程推进。
3. 拖动地图滑条远望后方节点，确认当前地点不改变、未来节点不可点击且不显示审稿章。
4. 确认地图图例解释 6 种节点分类标志。
5. 走完一个地点流程：抵达、线索、批注、审稿 / 互动、总结。
6. 点击“完成本站，前往下一站”，确认下一节点解锁并成为当前地点。
7. 刷新页面确认存档恢复。
8. 打开档案馆确认完整资料仍可查。
9. 结束旅程查看结局。
10. 重置进度。
11. 检查桌面和移动视口、键盘导航、控制台错误。
