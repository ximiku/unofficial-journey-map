# 取经路线非官方注释

公开演示地址：[https://ximiku.github.io/unofficial-journey-map/](https://ximiku.github.io/unofficial-journey-map/)

《取经路线非官方注释》是一部浏览器端互动文学游戏 / 数字人文作品。它从《西游记》取经路线出发，却不满足于把长安、五行山、火焰山、灵山排成一条干净的胜利路线。玩家扮演临时档案员，被要求重绘一张从长安到灵山的取经图；真正的任务，是决定那些被路线省略的无名随从、白龙马口供、火焰山灾后记录、狮驼岭亡者名册和无字经退件意见，究竟应该进入正文、留在边栏、藏入隐档、交给审稿，还是被保留为沉默。

这不是一个普通的章节浏览器，而是一张会被批注、审稿和留白不断改写的文学地图。地图越清楚，越可能删去复杂生命；旁注越完整，越可能让“可交付”的路线失去稳定形状。玩家在每一站做出的编辑选择，会改变正本度、旁注度、余声、驳杂、审稿风险、留白感、慈悲和路线清晰度，并导向不同结局。

## 游玩方式

1. 点击“开始整理”，进入取经路线复核案。
2. 按每站流程阅读抵达故事、查看线索、选择批注、处理审稿或互动。
3. 完成当前地点后进入下一站；未来地点可以在地图上远望，但不能直接跳关。
4. 随时进入档案馆查看已解锁节点、证据、声音、批注和审稿历史。
5. 在路线完成或提前结束时生成结局报告。

文档重点说明作品立意、文化问题意识、玩家身份、互动机制、视觉语言和课程展示价值，并附有游玩示意截图。

## 项目内容

- 26 处取经路线节点。
- 70+ 张非官方批注卡。
- 18+ 条审稿 / 审查 / 官僚批改事件。
- 8 个文学性结局。
- 多个小型互动模块，包括遗物整理、紧箍合约、名字陷阱、风向谜题、亡者名册和无字经校验。
- 档案馆、存档 / 重置、presentation mode 与 GitHub Pages 自动部署。

## 本地运行

```bash
npm install
npm run dev
```

开发服务器默认使用 `http://127.0.0.1:5173`。作品不需要后端，构建后可作为静态站点运行。

## 构建与测试

```bash
npm run typecheck

```

## 内容扩展

- 新增路线节点：编辑 `src/data/nodes.ts`。
- 新增批注卡：编辑 `src/data/annotations.ts`。
- 新增审稿事件：编辑 `src/data/reviews.ts`。
- 新增结局：编辑 `src/data/endings.ts`。
- 新增互动模块：编辑 `src/data/miniGames.ts` 与 `src/miniGames/index.tsx`。

## 已知限制

- 首版批注放置采用点击式交互，不做拖拽，以保证移动端和测试稳定性。
- 资料引用以章节和研究线索为主，未嵌入长篇原文。
- 互动模块优先表达文学机制，不追求动作游戏复杂度。
